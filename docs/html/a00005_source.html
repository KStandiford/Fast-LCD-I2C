109 /*
110 * For Arduino, the I2C interface (Wire) has an internal buffer
111 * The buffer length is defined as BUFFER_LENGTH.
112 * Care MUST be taken, since ONLY BUFFER_LENGTH characters can be sent
113 * in a singe transmission, and excess characters will be discarded.
114 * Note that BUFFER_LENGTH varies and can be quite short. UNO is about 30
115 * characters, while the Pi Pico implementation is over 120!
116 *
117 * In the Pi Pico SDK, the I2C interface does not buffer, so we can set the
118 * size to suit ourselves.
119 *
120 * Note that we will ALWAYS buffer internally and transmit in a block, since
121 * testing with Arduino showed that it was faster than sending single bytes
122 * to the Arduino Wire routines.
123 * */
