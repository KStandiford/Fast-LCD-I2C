119 /*
120 * For Arduino, the I2C interface (Wire) has an internal buffer
121 * The buffer length is defined as BUFFER_LENGTH.
122 * Care MUST be taken, since ONLY BUFFER_LENGTH characters can be sent
123 * in a singe transmission, and excess characters will be discarded.
124 * Note that BUFFER_LENGTH varies and can be quite short. UNO is about 30
125 * characters, while the Pi Pico implementation is over 120!
126 *
127 * In the Pi Pico SDK, the I2C interface does not buffer, so we can set the
128 * size to suit ourselves.
129 *
130 * Note that we will ALWAYS buffer internally and transmit in a block, since
131 * testing with Arduino showed that it was faster than sending single bytes
132 * to the Arduino Wire routines.
133 * */
