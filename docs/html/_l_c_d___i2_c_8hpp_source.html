117 /*
118 * For Arduino, the I2C interface (Wire) has an internal buffer
119 * The buffer length is defined as BUFFER_LENGTH.
120 * Care MUST be taken, since ONLY BUFFER_LENGTH characters can be sent
121 * in a singe transmission, and excess characters will be discarded.
122 * Note that BUFFER_LENGTH varies and can be quite short. UNO is about 30
123 * characters, while the Pi Pico implementation is over 120!
124 *
125 * In the Pi Pico SDK, the I2C interface does not buffer, so we can set the
126 * size to suit ourselves.
127 *
128 * Note that we will ALWAYS buffer internally and transmit in a block, since
129 * testing with Arduino showed that it was faster than sending single bytes
130 * to the Arduino Wire routines.
131 * */
