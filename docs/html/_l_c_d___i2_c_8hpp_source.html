118 /*
119 * For Arduino, the I2C interface (Wire) has an internal buffer
120 * The buffer length is defined as BUFFER_LENGTH.
121 * Care MUST be taken, since ONLY BUFFER_LENGTH characters can be sent
122 * in a singe transmission, and excess characters will be discarded.
123 * Note that BUFFER_LENGTH varies and can be quite short. UNO is about 30
124 * characters, while the Pi Pico implementation is over 120!
125 *
126 * In the Pi Pico SDK, the I2C interface does not buffer, so we can set the
127 * size to suit ourselves.
128 *
129 * Note that we will ALWAYS buffer internally and transmit in a block, since
130 * testing with Arduino showed that it was faster than sending single bytes
131 * to the Arduino Wire routines.
132 * */
